NoLifeChatter

A modular Twitch bot that translates live messages, searches a local SQLite chat archive, and experiments with Markov, retrieval, LoRA, and embedding-based persona tools.

Runtime shape

From one chat line to one careful reply

The public repo shows the system architecture without publishing the private archive. Commands share the same local database, while LLM features can stay pointed at LM Studio or any OpenAI-compatible local endpoint.

1

2

Gate

Anonymized visualization

A map of voices, not a leak of names

This network is a representative, anonymized sketch of what the archive analysis is for: finding nearby voices, active clusters, and odd edges worth inspecting. Labels and values are rounded placeholders.

What the map can answer

  • Which chatters cluster by language, topics, or posting rhythm?
  • Which accounts look unusually close after removing common phrases?
  • Which persona needs more history before it stops sounding generic?
  • Which retrieval examples are steering a generation?

Control dials

The interesting part is probability

A good bot should not answer everything. It needs confidence thresholds, retrieval scores, randomness, and posting limits that can be tuned without rewriting the command layer.

Directed command always
Retrieval confidence high

Sanitized examples

Commands without publishing private chat

The real bot works against real local logs. These examples use fake names and invented lines so the public page can show behavior without exposing the archive. 

user_07: ~said chatter_12 "try it"
bot: chatter_12 said that 14 times - first in 2025.
user_03: ~persona chatter_04 thoughts on this patch?
bot: [persona] chatter_04: it works but the vibes are suspicious
user_11: ~vibes chatter_19
bot: closest voices: chatter_06, chatter_22, chatter_31
user_02: ~regex anyone "\\bseason [0-9]+\\b"
bot: found matches across the local archive.

Public boundary

The page is public. The archive is not.

GitHub Pages only gets code, docs, and sanitized visuals. Local config, tokens, databases, private logs, rosters, smoke tests, and handoff notes that contain real usernames stay ignored or private.
